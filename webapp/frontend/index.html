Transit Configuration
GTFS feeds and transit network mapping settings
Enable Transit Network ? If enabled, uses pt2matsim pipeline to generate a multimodal network (road + transit) from OSM + GTFS. If disabled, road-only network.
GTFS Feed Settings
Catalog Max Age (days) ? Re-download the Mobility Database catalog CSV if the local copy is older than this many days.
Feed Max Age (days) ? Re-download individual GTFS feed zips if the cached copy is older than this many days.
GTFS Sample Day ? Which day to sample GTFS schedules for. 'dayWithMostTrips' auto-selects the busiest service day.

Controls how GTFS transit routes are mapped to the road network. Defaults work well for most regions.

Candidate Distance (m) ? Maximum distance to search for candidate network links near a transit stop. Increase for suburban/rural areas.
Max Travel Cost Factor ? Maximum allowed cost ratio for mapped vs shortest path. Higher = more permissive routing. Range: 1.5 to 5.0.
Link Threshold ? Minimum candidate links to find before expanding search radius.
Candidate Distance Multiplier ? When fewer than nLinkThreshold candidates are found, the search radius is expanded by this multiplier.

Scaling Parameters
Scaling Factor ? Fraction of full population to simulate. E.g., 0.10 = 10% sample. Lower values run faster but with less detail.
0.105

Target Plans ? 'all' generates plans for the entire scaled population. Or set a specific number to cap total plans.
Random Seed ? Random seed for reproducibility. Same seed = same plans each run.
Work Scaling Multiplier ? Multiplier applied on top of scaling_factor for work trips only. 1.0 = same as scaling_factor.
Activity Chain Settings
Min Chain Length ? Minimum activities per chain. 3 = smallest valid chain (Home-X-Home).
Max Work Activities ? Maximum Work activities per chain. 1 = simple commute, 2 = allows split-shift or lunch-at-home.
Early Stop Exponent ? Controls chain generation stopping. 1.0 = linear, 2.0 = quadratic (default). Higher = longer chains.

Simulation
MATSim parameters, traffic counts, and evaluation settings
MATSim Parameters
Iterations ? Number of MATSim iterations for route/plan optimization. More iterations = better convergence but longer runtime. 50-200 typical.
Link Stats Averaging ? Number of iterations to average link statistics over. Smooths out iteration-to-iteration noise.
Flow Capacity Factor ? Should roughly match scaling_factor. Controls how many vehicles can pass a link per time unit.
Storage Capacity Factor ? Should be >= flow_capacity_factor. Controls how many vehicles can be on a link simultaneously.
Traffic Counts
FHA Data Year ? Year of FHA/TMAS traffic count data to use for validation.
FHA Data Month ? Month of volume data (1-12). FHA distributes one zip per month.
